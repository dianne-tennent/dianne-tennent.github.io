Inline VS Inline-Block

Inline

When elements display inline, they sit within a line on the page and allow other elements to sit next to them on either side. You cannot adjust the width or height of an inline element, and margins will only apply to the left and right of the element(Padding will still apply).

An important thing to remember is that an inline element cannot contain a block-level element.

Examples of elements that display inline by default are:

  • <img>
  • <a>
  • <input>
  • <span>
  • <time>

Block

When elements display as "block", this means that they will be the only element that displays on that line. No other elements can sit beside them. Block elements allow you to adust their width and height, and margin/padding will apply on all sides.

Examples of elements that display as "block" by default are:

  • <div>
  • <main>
  • <h>
  • <article>
  • <aside>
  • <li>
  • <p>

Inline-Block

With inline-block, you basically have an inline element that allows other elements to sit next to it, while still being able to adjust the width and height, and also add margins to all sides.

A common use of inline-block is when you want buttons to display side by side, but still be able to add formatting to them.

Another useful application is where you have a block level element but you want other elements to sit next to it. It basically does what the float property does.
